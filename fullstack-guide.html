Full-stack, without the fog.

A practical walk through everything a beginner needs to build a real web app — frontend, backend in .NET / C#, and a SQL database. With examples you can read, run, and remember.

// 01What does "full-stack" actually mean?

A web application has three jobs happening at the same time:

  1. Show something pretty in the browser (frontend)
  2. Run logic, rules, and security on a server (backend)
  3. Remember things between visits (database)

A full-stack developer is someone who can work across all three. You don't need to be an expert in every layer — you need to understand how they talk to each other.

★ Mental model
Think of a restaurant. The menu and dining room = frontend. The kitchen = backend. The pantry/fridge = database. The waiter carrying messages between them = HTTP.

// 02The three layers, visually

F

Frontend — what the user sees

Runs in the browser. Built with HTML (structure), CSS (style), JavaScript (behavior). Frameworks: React, Vue, Angular, Blazor.

B

Backend — the rules and logic

Runs on a server. Handles auth, validation, business rules. Built here with ASP.NET Core + C#. Exposes an HTTP API.

D

Database — the memory

Stores data permanently. We'll use SQL Server (or any relational DB). The backend talks to it; the frontend never does directly.

⚠ Critical rule
The frontend never talks to the database directly. Always frontend → backend → database. This protects your data and lets you change either side without breaking the other.

// 03How the web talks: HTTP

Every interaction between frontend and backend is an HTTP request. The frontend asks; the backend answers. Each request has a method (the verb) and a URL (the noun).

MethodPurposeExample
GETRead dataGET /api/todos
POSTCreate new dataPOST /api/todos
PUTUpdate existing dataPUT /api/todos/5
DELETERemove dataDELETE /api/todos/5

Status codes — the backend's reply

Every response carries a number that says how it went:

200 OKWorked. Here's your data.
201 CreatedNew thing was made.
400 Bad RequestYour input was wrong.
401 UnauthorizedYou're not logged in.
404 Not FoundThat thing doesn't exist.
500 Server ErrorBackend crashed. Not your fault.

// 05Calling the backend from the browser

The frontend uses fetch() to send HTTP requests. Here's how to GET a list of todos and POST a new one:

JavaScript// Ask the backend for all todos
async function loadTodos() {
  const response = await fetch("/api/todos");
  const todos = await response.json();   // parse JSON body
  console.log(todos);   // e.g. [{id:1, title:"Buy milk"}]
}
JavaScript// Send a new todo to the backend
async function addTodo(title) {
  const response = await fetch("/api/todos", {
    method: "POST",                                // the verb
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title: title })  // payload as JSON
  });
  const created = await response.json();
  return created;
}
★ Key idea
Data flows as JSON — a text format both JavaScript and C# understand. The backend turns C# objects into JSON; the frontend turns JSON into JS objects. Same shape, different language.

// 06Backend with ASP.NET Core

ASP.NET Core is Microsoft's framework for building web APIs and apps in C#. It's free, cross-platform, and runs happily on your MacBook Pro M4.

Setting up — the four commands you need

terminal# 1. Check .NET is installed (should be 8 or later)
dotnet --version

# 2. Create a new Web API project
dotnet new webapi -n TodoApi

# 3. Move into it and run
cd TodoApi
dotnet run

# 4. Open the URL shown in the terminal (often https://localhost:5001) and append /swagger

The minimum file: Program.cs

C#// Program.cs — entry point of every ASP.NET Core app
var builder = WebApplication.CreateBuilder(args);

// Register the services your app needs
builder.Services.AddControllers();           // enable controllers
builder.Services.AddEndpointsApiExplorer();  // for Swagger
builder.Services.AddSwaggerGen();

var app = builder.Build();

// Set up the request pipeline (middleware order matters)
if (app.Environment.IsDevelopment()) {
    app.UseSwagger();
    app.UseSwaggerUI();
}
app.UseHttpsRedirection();
app.MapControllers();   // hook up controller routes

app.Run();   // start the web server

// 07Controllers — where your endpoints live

A controller is a C# class that handles HTTP requests for a specific resource. Each method = one endpoint.

C#// Controllers/TodosController.cs
using Microsoft.AspNetCore.Mvc;

[ApiController]                           // marks this as a Web API controller
[Route("api/[controller]")]              // URL becomes /api/todos
public class TodosController : ControllerBase
{
    // In-memory list (we'll swap for a real DB later)
    private static List<Todo> _todos = new() {
        new Todo { Id = 1, Title = "Learn full-stack" }
    };

    [HttpGet]                                // GET /api/todos
    public IActionResult GetAll() => Ok(_todos);

    [HttpGet("{id}")]                         // GET /api/todos/5
    public IActionResult GetOne(int id) {
        var todo = _todos.FirstOrDefault(t => t.Id == id);
        return todo == null ? NotFound() : Ok(todo);
    }

    [HttpPost]                               // POST /api/todos
    public IActionResult Create([FromBody] Todo input) {
        input.Id = _todos.Count + 1;
        _todos.Add(input);
        return CreatedAtAction(nameof(GetOne), new { id = input.Id }, input);
    }

    [HttpPut("{id}")]                         // PUT /api/todos/5
    public IActionResult Update(int id, [FromBody] Todo input) {
        var existing = _todos.FirstOrDefault(t => t.Id == id);
        if (existing == null) return NotFound();
        existing.Title = input.Title;        // copy editable fields
        existing.IsDone = input.IsDone;
        return NoContent();
    }

    [HttpDelete("{id}")]                      // DELETE /api/todos/5
    public IActionResult Delete(int id) {
        _todos.RemoveAll(t => t.Id == id);
        return NoContent();
    }
}

// The model — a plain C# class (sometimes called a POCO)
public class Todo {
    public int Id { get; set; }
    public string Title { get; set; } = "";
    public bool IsDone { get; set; }
}
What does each attribute do?
  • [ApiController] — turns on automatic JSON binding and 400 responses for bad input
  • [Route("api/[controller]")] — [controller] is replaced by the class name minus "Controller", so TodosController → /api/todos
  • [HttpGet], [HttpPost] etc — map a method to a verb
  • [FromBody] — read the value from the request body (as JSON) instead of the URL

// 08Dependency Injection — the .NET superpower

Instead of creating objects with new, you ask ASP.NET Core for them. This makes code testable and swappable.

C#// Step 1: Define an interface (a contract)
public interface ITodoService {
    List<Todo> GetAll();
}

// Step 2: Implement it
public class TodoService : ITodoService {
    public List<Todo> GetAll() => /* fetch from DB */ new();
}

// Step 3: Register it in Program.cs
builder.Services.AddScoped<ITodoService, TodoService>();

// Step 4: Use it in a controller (constructor injection)
public class TodosController : ControllerBase {
    private readonly ITodoService _service;
    public TodosController(ITodoService service) {
        _service = service;   // .NET passes it in automatically
    }
    [HttpGet]
    public IActionResult GetAll() => Ok(_service.GetAll());
}
★ Three lifetimes
  • Singleton — one instance for the whole app
  • Scoped — one per HTTP request (most common)
  • Transient — a new one every time it's asked for

// 09SQL — talking to the database

SQL is the language for relational databases. Data lives in tables (rows and columns, like a spreadsheet). Each table has a primary key — a unique ID per row.

Creating a table

SQL-- Create a Users table with three columns
CREATE TABLE Users (
    Id        INT PRIMARY KEY IDENTITY(1,1),  -- auto-increment
    Name      NVARCHAR(100) NOT NULL,
    Email     NVARCHAR(200) UNIQUE NOT NULL,
    CreatedAt DATETIME DEFAULT GETDATE()
);

The four operations (CRUD)

SQL-- Insert a new row
INSERT INTO Users (Name, Email)
VALUES ('Anon', 'anon@example.com');
SQL-- Read everything
SELECT * FROM Users;

-- Read with a filter
SELECT Name, Email FROM Users
WHERE Email LIKE '%@example.com'
ORDER BY CreatedAt DESC;
SQL-- Update a row (always include WHERE!)
UPDATE Users
SET Name = 'Anonymous'
WHERE Id = 1;
SQL-- Delete a row
DELETE FROM Users
WHERE Id = 1;
⚠ The painful lesson
Always include WHERE with UPDATE and DELETE. Without it, you change every row in the table. Test on SELECT first.

// 10Joins — combining tables

Real apps have many related tables. A JOIN stitches them together using a shared key.

The setup: two tables

SQL-- Each Todo belongs to one User (UserId is the foreign key)
CREATE TABLE Todos (
    Id     INT PRIMARY KEY IDENTITY(1,1),
    Title  NVARCHAR(200) NOT NULL,
    UserId INT NOT NULL FOREIGN KEY REFERENCES Users(Id)
);

INNER JOIN — only matching rows

SQL-- Get every todo together with the owner's name
SELECT t.Title, u.Name
FROM Todos t
INNER JOIN Users u ON t.UserId = u.Id;

LEFT JOIN — keep all rows from the left side

SQL-- Every user, even those with zero todos
SELECT u.Name, t.Title
FROM Users u
LEFT JOIN Todos t ON t.UserId = u.Id;
What is "normalization"?

Splitting data into multiple tables to avoid repetition. If we kept the user's name in every Todo row, changing the name would mean updating many rows. By keeping users in their own table and using a foreign key, we store each fact once.

  • 1NF: each cell holds a single value (no comma-separated lists)
  • 2NF: non-key columns depend on the whole primary key
  • 3NF: non-key columns depend only on the key, nothing else

// 12Entity Framework Core — C# meets SQL

Writing raw SQL from C# is tedious. EF Core is an ORM (Object-Relational Mapper) — it lets you work with C# objects, and translates to SQL behind the scenes.

The four-step pattern

terminal# Install EF Core packages
dotnet add package Microsoft.EntityFrameworkCore.SqlServer
dotnet add package Microsoft.EntityFrameworkCore.Design
C#// Step 1: Define your entity (matches a table)
public class Todo {
    public int Id { get; set; }
    public string Title { get; set; } = "";
    public bool IsDone { get; set; }
}

// Step 2: Create a DbContext (your bridge to the DB)
public class AppDbContext : DbContext {
    public AppDbContext(DbContextOptions<AppDbContext> opts) : base(opts) {}

    // Each DbSet<T> becomes a table
    public DbSet<Todo> Todos => Set<Todo>();
}

// Step 3: Register it in Program.cs (with connection string)
builder.Services.AddDbContext<AppDbContext>(opts =>
    opts.UseSqlServer(builder.Configuration.GetConnectionString("Default"))
);

// Step 4: Inject it into a controller and query
public class TodosController : ControllerBase {
    private readonly AppDbContext _db;
    public TodosController(AppDbContext db) => _db = db;

    [HttpGet]
    public async Task<IActionResult> GetAll() {
        var todos = await _db.Todos.ToListAsync();   // SELECT * FROM Todos
        return Ok(todos);
    }

    [HttpPost]
    public async Task<IActionResult> Create([FromBody] Todo todo) {
        _db.Todos.Add(todo);
        await _db.SaveChangesAsync();              // INSERT INTO Todos
        return CreatedAtAction(nameof(GetAll), new { id = todo.Id }, todo);
    }
}

The connection string

JSON// appsettings.json — keep secrets out of source control in real projects
{
  "ConnectionStrings": {
    "Default": "Server=localhost;Database=TodoApp;Trusted_Connection=True;TrustServerCertificate=True;"
  }
}

Migrations — keeping schema in sync

terminal# Generate a migration from your C# models
dotnet ef migrations add InitialCreate

# Apply it to the database (creates tables)
dotnet ef database update
★ On macOS / Apple Silicon
SQL Server runs natively only via Docker on Apple Silicon. For local dev, many .NET devs on Mac use SQLite (replace UseSqlServer with UseSqlite) or PostgreSQL (Npgsql package). EF Core works with all of them — same C# code, different provider.

// 13Mini project: a Todo API end-to-end

Putting every piece together. Imagine a user typing in a task in the browser:

The journey of one click

  1. User types "Buy groceries" and clicks Save in the browser.
  2. Frontend JS calls fetch("/api/todos", { method:"POST", body:... }).
  3. ASP.NET Core receives the request, routes it to TodosController.Create.
  4. Controller validates input, asks AppDbContext to add the row.
  5. EF Core generates INSERT INTO Todos… and runs it.
  6. SQL Server stores the row, returns the new Id.
  7. Controller returns 201 Created with the saved object as JSON.
  8. Frontend reads the JSON, updates the on-screen list. Done.

Try it yourself — the build order

  1. dotnet new webapi -n TodoApi
  2. Add EF Core packages (SQLite is easiest on Mac M4)
  3. Create Todo.cs model + AppDbContext.cs
  4. Wire up DI in Program.cs
  5. Create TodosController.cs with GET / POST / PUT / DELETE
  6. Run dotnet ef migrations add Init + dotnet ef database update
  7. Open Swagger UI and test each endpoint
  8. Build a tiny index.html page that calls the API with fetch
